Thoughts on CoffeeScript

Feb 24th, 2013 • Permalink

I’ve been working through Noel Rappin’s Mastering Space And Time With JavaScript to level-up my JavaScript abilitites, converting the examples to CoffeeScript as I go along.

I’ve been aware of CoffeeScript for a while but this is the first I’ve used it for anything substantial, and it’s been interesting experience.

I’m using the jasminerice gem which integrates well into the Rails assett pipeline, so you never need to think about compiling.

CoffeeScript is the first language I’ve used with significant whitespace, and I’m quite liking it. No need to waste a line of vertical space for simple end statement.

I’m often surprised by how few brackets and commas I need, and the ease of spreading things out over multiple lines:

In Ruby this would look something like:

“` ruby

product: "alpha",
company: "beta"

}

  • The code I’ve converted from JS to CS ends up about 30 - 40% shorter.

I find the noise from the brackets, braces and semi-colons in JS really distracting. Especially when you have callbacks involved.

Things that I get wrong:

  • As with JS, when you call a method you need the parenthesis after the method name.

Things I’m still getting used to:

Jasmine uses spies rather than mocks. Although spies are a kind of mock, I still need to think hard about them.

CoffeeScript doesn’t mean you can forget all JS’s scoping rules.
